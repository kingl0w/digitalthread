# Design Context — Digital Thread

## Target audience
Engineers, data people, and hiring managers evaluating this as a portfolio project. Secondary:
aviation-data curious visitors. They land cold from a GitHub link and should grasp the system in
30 seconds without reading the README.

## Use cases
- See what the system does (fragmented FAA sources → one governed graph → two killer queries)
  and watch it happen live against the real database.
- Judge the engineering: real data, verifiable numbers, honest about what's synthetic.
- Jump off to GraphiQL (the console) or the GitHub repo for depth.

## Brand personality / tone
"Living airworthiness directive": editorial-industrial, aviation paperwork brought to life.
Federal Register meets instrument placard — authoritative, precise, paper-first, quietly warm.
Emphatically NOT a SaaS dashboard and NOT dark-mode-neon. Real serials and N-numbers are the
texture; monospace appears only where actual data appears.

- Type: Archivo Variable (condensed placard display ↔ expanded stamps), Spline Sans Mono for
  data only.
- Palette: warm paper + near-black blue ink; safety-orange accent (inspection tags); green =
  airworthy, signal red = failures, amber = material lots. Light-first; dark mode supported.
- The one memorable thing: pressing a stamp-like button executes the real traversal and the
  affected fleet resolves in front of you, N-number by N-number, with the latency printed like
  a teletype line.
